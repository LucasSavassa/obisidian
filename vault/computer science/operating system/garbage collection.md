- garbage collection is one type of automatic memory management
# memory usage
- programs request memory to use
- operating system scan for free memory
- operating system reserves some free memory
- operation system returns memory address
- other programs can't access reserved memory
![[garbage collection 2025-12-17 10.49.24.excalidraw]]
- if a program doesn't release memory
- then memory runs out
- this is called **memory leaking**
- therefore, garbage collection exist
